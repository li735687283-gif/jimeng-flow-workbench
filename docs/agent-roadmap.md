# 即梦 Flow 画布 Agent 功能升级路线图

> 历史规划快照（2026-07-05），不作为当前能力清单。

> 基于竞品调研（即梦AI、Dify、n8n、Flowise、Langflow、ChatGPT Canvas）整理，结合项目现有技术栈制定。

---

## 调研结论：其他画布/工作流工具的核心 Agent 功能

| 工具 | 核心 Agent 功能 |
|------|---------------|
| **即梦AI（无限画布）** | 故事模式→分镜批量生成→图片→视频链；角色一致性锁定；多风格并行生成；智能画布编辑（局部重绘/扩图/消除）；首尾帧控制；镜头运动控制；Agent 自动执行（用户说"可以"即可） |
| **Dify** | 多 Agent 协同（ReAct + Function Calling）；条件分支/循环/子流程；RAG 知识库；Prompt IDE；MCP 协议支持 280+ 外部工具；实时日志监控 |
| **n8n** | 500+ 集成节点；AI Agent 作为工作流节点；Agent 可调用其他 Agent（Tool Node）；执行历史/逐节点调试 |
| **Flowise** | 多模态处理（CV+NLP+LLM）；可视化节点编排；API 管理；实时执行监控；插件扩展系统 |
| **Langflow** | 对话式 AI 工作流；场景编辑器；文档检索；图像生成；代码节点嵌入 |
| **CanvasMind** | 多模式切换（Agent/图片/视频）；SSE 实时推送；技能管理/模型管理；模板插入；版本回滚 |
| **ChatGPT Canvas** | 创作伙伴模式（批注+建议）；协作式编辑；代码 Review 标注 |

## 升级路线图（分 5 个阶段，按优先级排序）

### 阶段 1：Agent 对话体验重构（对话流 + 智能执行）

> **目标**：让 Agent 的对话流程更智能、更丝滑，参考即梦AI"输入→思考→确认→执行"的完整链路。

1. **状态化对话流**（本次已实现）
   - ✅ 思考过程可折叠展示
   - ✅ 技能加载状态（加载技能→生成图片→生成视频）
   - ✅ 图片/视频确认卡片
   - ✅ 根据 LLM 意图自动选择技能

2. **Agent 自动执行（无需重新输入）**
   - 用户说"可以""好""确认"→ 直接继续执行（如图片生成完后自动触发视频）
   - 用户说"不""算了"→ 取消当前技能，回到对话状态
   - 实现方式：在 submit 中检测回复关键字，自动路由到 pending 请求的继续执行

3. **对话上下文记忆（同一会话内）**
   - 用户说"把那张图改成竖屏"→ Agent 知道"那张图"指上一步生成结果
   - 用户说"再生成一个类似风格的"→ 自动复用之前的提示词和参数
   - 实现方式：在 agentStore 中维护 `conversationContext` 对象，记录最近的操作和结果

4. **一键继续/重试**
   - 在失败或取消后，消息下方显示"重试""换个模型"按钮
   - 在生成完成后显示"继续生成视频""生成更多图片"按钮

---

### 阶段 2：批量生成与风格一致性（多结果+参考图）

> **目标**：从单图生成升级为批量创作，支持多结果横向排列和风格锁定。

5. **批量生成 N 张图**（已部分实现，扩展为批量）
   - 用户说"生成4张不同角度的场景图"
   - Agent 在后台拆分为4个并行生成任务，结果横向排列在画布上
   - 实现方式：支持一次提交 `count: 4`（即梦 CLI 本身支持）

6. **角色/风格一致性锁定**
   - 用户上传参考图或生成第一张图后，后续生成自动复用参考图保持风格一致
   - 显示"已锁定参考图"提示，可一键解除
   - 实现方式：在 `conversationContext` 中保存 `referenceAssetId`，后续生成请求自动加入 `inputImages`

7. **分镜/故事模式**
   - 用户输入"帮我做一个30秒中秋月饼带货视频"
   - Agent 自动拆解为：剧本 → 分镜列表（6个镜头）→ 批量生成图片 → 确认后批量生成视频
   - 实现方式：后端 SYSTEM_PROMPT 新增 `storyMode` 意图，返回 `storyboard` 数组（每个元素包含镜头描述、画面描述），前端逐一显示并批量生成

8. **多风格并行生成**
   - 用户说"试试水墨风格和赛博朋克风格"
   - Agent 生成两个不同风格的版本，并排展示
   - 实现方式：SYSTEM_PROMPT 返回 `variants` 数组，每个 variant 对应一套参数

---

### 阶段 3：图片编辑技能（智能画布编辑）

> **目标**：让 Agent 不仅能生成内容，还能修改已有内容。

9. **图片局部重绘（In-painting）**
   - 用户选中画布上某张图片，对 Agent 说"把人物的衣服换成红色"
   - Agent 调用即梦 CLI 的 inpainting 能力（或图生图 + mask）
   - 实现方式：在 AgentPanel 中支持"引用图片+编辑指令"模式，后端生成 mask 区域描述

10. **图片扩图（Out-painting）**
    - 用户说"把这张图左右扩展一些，做成横屏"
    - Agent 自动调整尺寸并扩展画面

11. **图片消除/抠图**
    - 用户说"去掉背景中多余的人""把主体抠出来"
    - 调用 CLI 的消除或抠图功能

12. **图生图风格迁移**
    - 用户上传一张图，说"把这张转成油画风格"
    - 用 reference 图 + 风格提示词重新生成

---

### 阶段 4：多 Agent 协同与记忆（高级编排）

> **目标**：从单一 Agent 升级为多角色协作，支持复杂创作任务。

13. **多 Agent 角色模式**
    - 导演 Agent：负责整体创意和调度
    - 编剧 Agent：负责剧本和分镜
    - 视觉 Agent：负责图片生成和风格控制
    - 剪辑 Agent：负责视频生成和节奏
    - 用户选择"故事模式"后，系统自动切换多 Agent 协作

14. **对话持久化记忆（跨会话）**
    - 用户关闭面板后重新打开，Agent 记得之前的创作上下文
    - 实现方式：将 `conversationContext` 保存到 localStorage / 后端

15. **知识库/RAG 支持**
    - 用户上传参考资料（风格文档、品牌指南），Agent 在生成时参考
    - 实现方式：前端上传文件，后端提取文本存入向量存储，SYSTEM_PROMPT 中注入相关知识

16. **外部工具集成（MCP 协议）**
    - Agent 可以调用外部 API（如搜索图片、获取天气、调用其他 AI 服务）
    - 实现方式：通过后端 MCP 客户端，支持标准的 MCP 协议调用

---

### 阶段 5：实时流与体验优化（工程完善）

> **目标**：提升实时性和交互体验。

17. **SSE 实时推送生成进度**
    - 后端生成过程中实时推送进度（5%→20%→80%→100%）
    - 前端显示实时进度条和中间结果预览
    - 替代当前的"生成中"轮询状态

18. **生成结果预览/画廊**
    - 生成完成后，在 Agent 面板内直接预览图片/视频
    - 支持左右滑动浏览多结果

19. **语音指令交互**
    - 长按语音按钮，直接口述复杂需求
    - 语音识别 + 语义理解后自动执行
    - 语音已部分实现，扩展为支持自然语言指令的完整执行链

20. **模板系统**
    - 用户说"创建一个电商产品视频"，Agent 加载预设模板（开场→产品展示→卖点→结尾）
    - 模板可以保存和复用

## 实现建议

### 推荐优先实现的 Top 5 功能（按性价比排序）

| 排名 | 功能 | 预估工作量 | 价值 |
|------|------|-----------|------|
| 1 | Agent 自动执行（"确认/取消"关键词） | 1-2 天 | 极高，让流程更丝滑 |
| 2 | 对话上下文记忆（同会话内） | 2-3 天 | 高，大幅减少重复输入 |
| 3 | 分镜/故事模式 | 3-4 天 | 极高，即梦AI的核心卖点 |
| 4 | 角色/风格一致性锁定 | 2-3 天 | 高，批量创作刚需 |
| 5 | 生成结果实时预览 | 2-3 天 | 中，提升体验 |

### 技术注意事项
- 当前 `SYSTEM_PROMPT` 已支持 `intent` 和 `thinking`，后续可在 `suggestedParams` 中扩展更多字段（如 `referenceStyle`、`storyboard` 等）
- 批量生成可通过即梦 CLI 的 `count` 参数直接实现，无需后端大改
- 图片编辑需要调研即梦 CLI 是否支持 inpainting/outpainting，可能需要额外集成
- 多 Agent 协同可通过在后端创建多个 `Agent` 服务实例，用不同的 `SYSTEM_PROMPT` 实现
